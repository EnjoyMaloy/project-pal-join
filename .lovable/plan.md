# Что меняю на `/my-courses`

## 1. Блок выбора уроков → дизайн карты курса (как на `/course/1/lessons`)

Заменяю текущий ряд цветных кружков на тот же визуал, что и на странице карты курса:

- Фон секции: фиолетовый градиент с точечным паттерном (radial-gradient точки).
- Сверху в углу — белая карточка «Пройдено N%» с тонким прогресс-баром (общий прогресс по курсу).
- Уроки — кружки с градиентом из снейк-карты:
  - **Пройден** → пурпурный градиент `gPurpleNode` + иконка трофея.
  - **Текущий** → золотой градиент `gGoldNode` + иконка-чеклист + плашка «Начать» под кружком.
  - **Заблокирован** → белый градиент `gWhiteNode` + замок.
- Расположение — компактная сетка/вертикальный «змейка»-маршрут, адаптированный под количество уроков (4 шт.).
- Бейдж «инструкция» (скрепка) сохраняется в углу кружка.
- Поповер с описанием урока при клике — оставляю текущую логику и стиль.

## 2. Открытый урок → оформление как статья (по референсу)

Перерабатываю структуру `lessonsData[i].content` и рендер:

- Заголовок H1 урока.
- Шапка-карточка `bg-muted rounded-2xl` под H1 с метаданными (автор-аватар, просмотры, время чтения, дата) — как в референсе.
- Тело: чередование
  - параграфов (`text-body-14`),
  - подзаголовков H2 (`Разберём на пальцах`, `Пример`),
  - жирных мини-заголовков перед абзацем (`Длинная позиция (лонг)`),
  - **жирных** инлайн-фрагментов внутри параграфа,
  - инлайн-«выделяшек» — violet pill `bg-[#E8DCFB] text-[#460466] rounded-md px-1.5`.
- Сохраняется: 2 картинки-плейсхолдера сверху и 2 снизу (как уже сделано).

Формат секции расширяю до:
```ts
type Inline = string | { text: string; bold?: boolean; highlight?: boolean };
type Section =
  | { type: "h2"; text: string }
  | { type: "h3"; text: string }
  | { type: "p"; runs: Inline[] }
  | { type: "list"; items: Inline[][] };
```

Контент уроков переписываю под этот формат с подходящими выделениями ключевых фраз.

## 3. Файлы

- `src/pages/Index.tsx` — единственный затронутый файл.
- `CourseLessons.tsx` не трогаю — оттуда только копирую SVG-узлы и стиль карточки прогресса.

## Технические детали

- SVG-узлы инкапсулирую в маленькие хелперы `<TrophyNode/>`, `<CurrentNode/>`, `<LockedNode/>` чтобы не дублировать пути.
- Highlight-pill — отдельный inline-компонент `<HL>{text}</HL>`.
- Никаких новых зависимостей, без правок типов перевода (`t()` не использую для новых строк).
