

## Техническое задание — план

Подготовлю подробный Markdown-документ (ТЗ) для команды разработки на основе прототипа. Документ будет сохранён в `/mnt/documents/`.

### Структура документа

1. **Введение** — цель документа, ссылка на прототип, общее правило: добавлять новый функционал, не заменяя существующий
2. **Новая сущность: Гайд (Instruction/Guide)** — описание модели данных, связь с курсом и уроком, единая сущность создаётся в студии курсов
3. **Раздел "Мои курсы" — новые элементы**
   - Скрепка на кружке урока (иконка Paperclip), если к уроку привязан гайд
   - Чип "Инструкция" в поповере урока при наличии гайда
   - Блок гайда внутри открытого урока (рядом с контентом урока)
   - Боковая панель "Инструкции из курса" (desktop) — список всех гайдов курса с возможностью чтения
4. **Раздел "Гайды" (новый пункт в сайдбаре)**
   - Поиск по гайдам (navbar)
   - Табы: Все / Избранное
   - Фильтры по категории (задаётся при создании гайда)
   - Сортировка: новые / популярные
   - Карточка гайда: градиент + 3D-картинка (настраиваемые), автор, охват, кнопки "скопировать ссылку" и "добавить в избранное"
   - Страница просмотра гайда
5. **Мобильная версия**
   - Bottom nav второго уровня: Назад, Курс, Гайды, Квест
   - В "Мои курсы" вкладка "Гайды" заменяет "Задания"
   - Мобильные карточки гайдов
6. **Темная и светлая тема** — все новые элементы должны поддерживать обе темы
7. **Локализация** — RU/EN для всех новых текстов интерфейса
8. **Примечание** — ТЗ на создание/редактирование гайдов в студии курсов будет предоставлено отдельно

### Технические детали

- Документ создаётся как Markdown-файл через `code--exec`
- Сохраняется в `/mnt/documents/technical-spec.md`
- Содержимое основано на анализе файлов: `Index.tsx`, `Instructions.tsx`, `ArticleView.tsx`, `BottomNav.tsx`, `Sidebar.tsx`, `Navbar.tsx`, `LanguageContext.tsx`
- Включу описание UI-компонентов, состояний, взаимодействий и ссылки на прототип
